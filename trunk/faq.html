Frequently Asked Questions (FAQ) on SNMP_Session.pm

When will SNMP_Session.pm support SNMPv3? 

> 	I came to know from Mr. Tobias Oetikar, that you are working
> on the SNMP library used in the MRTG. I would like to know, if there
> is any work going on to support the SNMP V3 in the future releases
> of MRTG. We would like to use the security features which SNMP V3
> supports.

SNMP_Session.pm would have to be modified to understand SNMPv3. Personally I think this would be stretching the current code too much, and that it would be better to start with a fresh code base.

Net::SNMP is an alternative library that started out looking a lot like SNMP_Session.pm, but is written in a much cleaner way and it already supports SNMPv3. Like SNMP_Session.pm, it is stand-alone in that it doesn't require an additional SNMP toolkit like UCD-SNMP/Net-SNMP.

It might be a useful exercise to modify MRTG to use Net::SNMP instead of my SNMP_Session.pm (and Mike Mitchell's SNMP_util.pm which comes with it). However, it might require some work to do this in a compatible way. 

> Do we have any 3rd party tools which can be integrated
> with MRTG for the SNMP V3 support ?

I often use Net-SNMP (the former UCD-SNMP) as a command-line tool to access router information using SNMPv3. I have my SNMPv3 USM credentials configured in my ${HOME}/.snmp/snmp.conf file, so I can simply call snmptable ROUTERNAME TABLENAME without giving a community/user name.

It used to be the case that MRTG allowed the usage of external SNMP command-line tools, including the CMU ones, which later became UCD-SNMP. Maybe this code still works?
